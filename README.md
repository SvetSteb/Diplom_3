# Diplom_3

## Дипломный проект. Задание 3: UI-тесты

### Автотесты для проверки части функционала сайта Stellar Burgers

### Реализованные сценарии

#### Восстановление пароля: **test_password_restore.py**
* 1. переход на страницу восстановления пароля по кнопке «Восстановить пароль» **test_restore_password_get_restore_page** 
* 2. ввод почты и клик по кнопке «Восстановить» - **test_restore_password**
* 3. клик по кнопке показать/скрыть пароль делает поле активным — подсвечивает его. - **test_visibility_of_password_field**

#### Личный кабинет: **test_account_page.py**
* 1. переход по клику на «Личный кабинет» **test_personal_account_for_auth_user** 
* 2. переход в раздел «История заказов» - **test_personal_account_order_history**
* 3. выход из аккаунта - **test_logout_user**

#### Проверка основного функционала: **test_main_page.py**
* 1. переход по клику на «Конструктор», переход по клику на «Лента заказов» (параметризован для проверки двух требований)  **test_button_on_main_page** 
* 2. если кликнуть на ингредиент, появится всплывающее окно с деталями **test_ingredient_info_modal_window**
* 3. всплывающее окно закрывается кликом по крестику - **test_close_ingredient_info**
* 4. при добавлении ингредиента в заказ, увеличивается каунтер данного ингредиента **test_put_ingredient_in_bascet**
* 5. залогиненный пользователь может оформить заказ

#### Раздел «Лента заказов»: **test_queue_page.py**
* 1. если кликнуть на заказ, откроется всплывающее окно с деталями,  **ttest_order_info_window_is_displayed** 
* 2. заказы пользователя из раздела «История заказов» отображаются на странице «Лента заказов» **test_order_number_in_histiry_and_queue**
* 3. при создании нового заказа счётчик Выполнено за всё время увеличивается,
при создании нового заказа счётчик Выполнено за сегодня увеличивается, - **test_counters_afret_order**
* 4. после оформления заказа его номер появляется в разделе В работе. **test_order_in_progress**

### Структура проекта

- `locators` - пакет, содержащий локаторы веб-элементов, раздлеленные по станицам сайта
- `methods` - пакет, содержащий методы для работы с веб-элементами страниц
- `tests` - пакет, содержащий тесты

### Запуск автотестов

> `$ pytest -v tests/ --alluredir=allure_results`

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск сервера allure и формирование отчета**

>  `$ allure serve allure_results`
